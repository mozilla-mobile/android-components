setMaxLevel

fun setMaxLevel(level: Priority, includePII: Boolean = false): Unit (source)

Set the maximum level of logs that will be forwarded to Log. By default, the max level is DEBUG.

This is somewhat redundant with Log.logLevel (and a stricter filter on Log.logLevel will take precedence here), however setting the max level here can improve performance a great deal, as it allows the Rust code to skip a great deal of work.

This includes a includePII flag, which allows enabling logs at the trace level. It is ignored if level is not Log.Priority.DEBUG. These trace level logs* may contain the personal information of users but can be very helpful for tracking down bugs.

Parameters

level - The maximum (inclusive) level to include logs at.

includePII - If level is Log.Priority.DEBUG, allow debug logs to contain PII.
